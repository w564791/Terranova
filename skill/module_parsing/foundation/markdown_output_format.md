## Markdown 输出格式规范

### 标题层级
- `#` 一级标题：文档标题（每个文档只能有一个）
- `##` 二级标题：主要章节
- `###` 三级标题：子章节
- `####` 四级标题：细分内容
- **禁止跳级使用标题**（如从 `##` 直接跳到 `####`）

### 表格格式
必须包含表头和分隔行，格式如下：

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

表格规则：
- 表头必须有意义，不能为空
- 分隔行使用至少3个连字符 `---`
- 单元格内容不换行
- 单元格内容中的 `|` 需要转义为 `\|`

### 代码块
- 必须使用三个反引号包裹
- 必须指定语言类型

支持的语言类型：
- `json` - JSON 数据
- `yaml` - YAML 配置
- `hcl` - Terraform HCL
- `bash` - Shell 命令
- `markdown` - Markdown 示例

示例：
```json
{
  "key": "value"
}
```

### 列表格式
- 无序列表使用 `-` 开头
- 有序列表使用 `1.` `2.` `3.` 开头
- 嵌套列表缩进2个空格

### 强调格式
- **粗体**：使用 `**文字**`
- *斜体*：使用 `*文字*`
- `代码`：使用反引号包裹

### 强制规则
1. 只输出 Markdown 格式内容
2. 不添加任何额外解释文字
3. 不使用 HTML 标签
4. 不使用 Markdown 扩展语法（如脚注、任务列表等）
5. 确保所有链接格式正确 `[文字](URL)`

### 错误处理
- 如果内容包含特殊字符，必须正确转义
- 如果表格数据为空，省略该表格而不是显示空表格
- 如果章节没有内容，省略该章节
