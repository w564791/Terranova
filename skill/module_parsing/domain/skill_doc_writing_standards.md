## Skill 文档写作规范

### 目标读者
Skill 文档有两类读者，需要同时满足他们的需求：

| 读者类型 | 优先级 | 需求 |
|----------|--------|------|
| AI 模型 | 主要 | 理解 Module 配置规则，生成正确的配置 |
| 人类开发者 | 次要 | 审核、维护、调试 Skill 文档 |

### 内容原则

#### 1. 准确性 (Accuracy)
- 所有信息**必须来自 Schema**，不能编造
- 字段类型、约束、默认值必须与 Schema 一致
- 如果 Schema 中没有某信息，不要猜测

#### 2. 完整性 (Completeness)
- 覆盖**所有必填字段**
- 覆盖**重要的可选字段**（有默认值或常用的）
- 说明字段间的关联关系
- 包含外部数据源映射

#### 3. 简洁性 (Conciseness)
- 每个字段描述**不超过一行**
- 避免冗余信息
- 使用表格而非长段落
- 省略没有内容的章节

#### 4. 可操作性 (Actionability)
- 最佳实践必须**具体可执行**
- 避免空泛的建议（如"注意安全"）
- 提供具体的配置值或范围

### 章节结构
Skill 文档应包含以下章节（按顺序）：

#### 1. 模块概述
- **长度**：2-3 句话
- **内容**：模块用途、适用场景
- **示例**：
  > 该模块用于在 AWS 上创建和管理 EC2 实例。支持标准实例和 Spot 实例，适用于各种计算资源部署场景。

#### 2. 参数约束
包含三个子章节：

**2.1 必填字段**
| 字段名 | 类型 | 验证规则 | 描述 |
|--------|------|----------|------|

**2.2 重要可选字段**
| 字段名 | 类型 | 默认值 | 描述 |
|--------|------|--------|------|

**2.3 参数关联关系**
- 条件必填关系
- 互斥关系
- 依赖关系

#### 3. 配置分组
按 UI 分组说明字段归属：
- **基础配置 (basic)**：核心必填字段
- **高级配置 (advanced)**：可选功能字段
- 其他分组...

#### 4. 数据源映射
| 字段 | 数据源 ID | API 路径 | 说明 |
|------|-----------|----------|------|

#### 5. 最佳实践
- 3-5 条具体建议
- 每条建议有明确的操作指导

#### 6. 输出属性
| 输出名 | 类型 | 描述 |
|--------|------|------|

### 表格编写规范

#### 必填字段表格
| 列名 | 内容来源 | 说明 |
|------|----------|------|
| 字段名 | `properties` 的 key | 原始字段名 |
| 类型 | `properties[field].type` | string/integer/boolean/array/object |
| 验证规则 | enum/pattern/min/max 等 | 多个规则用分号分隔 |
| 描述 | label 或 description | 不超过 50 字符 |

#### 可选字段表格
| 列名 | 内容来源 | 说明 |
|------|----------|------|
| 字段名 | `properties` 的 key | 原始字段名 |
| 类型 | `properties[field].type` | 数据类型 |
| 默认值 | `properties[field].default` | 如无默认值显示 `-` |
| 描述 | label 或 description | 不超过 50 字符 |

### 最佳实践编写规范

**好的最佳实践**：
- ✅ "生产环境建议保持 `disable_api_termination` 为 true，防止误删除"
- ✅ "`security_group_ids` 应遵循最小权限原则，只开放必要端口"
- ✅ "建议配置 `min_size` >= 2 实现高可用"

**差的最佳实践**：
- ❌ "注意安全配置"（太空泛）
- ❌ "根据需要配置"（没有指导意义）
- ❌ "参考官方文档"（没有具体内容）

### 质量检查清单
生成文档后，检查以下项目：

- [ ] 所有 `required` 数组中的字段都在必填表格中
- [ ] 枚举字段列出了所有可选值
- [ ] 字段描述来自 Schema，未编造
- [ ] 表格格式正确（有表头和分隔行）
- [ ] 最佳实践具有可操作性
- [ ] 没有空章节
- [ ] Markdown 格式正确
